## 🚀 本地部署（已完成）

### ✅ 服务器已启动

**访问地址**：
- 📱 本机：http://localhost:8000
- 📱 局域网：http://你的 IP:8000

**局域网设备访问**：
1. 确保手机/平板和电脑在同一 WiFi
2. 查看电脑 IP：`ifconfig | grep "inet "`
3. 手机浏览器访问：`http://电脑 IP:8000`

**停止服务器**：
```bash
# 找到进程
lsof -i :8000

# 杀死进程
kill -9 进程 ID
```

## 🌐 GitHub Pages 部署

### 步骤 1: 创建 GitHub 仓库

1. 登录 GitHub
2. 创建新仓库：`primary-accounting-exam`
3. 勾选 "Initialize this repository with a README"

### 步骤 2: 配置部署脚本

编辑 `deploy_github.sh`：
```bash
# 修改这两行
USER_NAME="your-github-username"  # 改为你的 GitHub 用户名
REPO_NAME="primary-accounting-exam"
```

### 步骤 3: 运行部署

```bash
cd /Users/bao/.openclaw/workspace/初级会计备考

# 添加执行权限
chmod +x deploy_github.sh

# 运行部署
./deploy_github.sh
```

### 步骤 4: 启用 GitHub Pages

1. 进入仓库 Settings
2. 找到 Pages 设置
3. Source 选择 `gh-pages` 分支
4. 保存后等待 1-2 分钟

### 步骤 5: 访问在线版

```
https://your-username.github.io/primary-accounting-exam/
```

## 📱 局域网访问配置

### 查看本机 IP

**macOS/Linux**:
```bash
ifconfig | grep "inet "
```

**Windows**:
```bash
ipconfig
```

### 防火墙设置

**macOS**:
1. 系统偏好设置 > 安全性与隐私
2. 防火墙 > 防火墙选项
3. 添加 Python 或允许传入连接

**Windows**:
1. 控制面板 > Windows Defender 防火墙
2. 高级设置 > 入站规则
3. 新建规则 > 允许端口 8000

## 🔧 故障排查

### 问题 1: 端口被占用

```bash
# 查看占用端口的进程
lsof -i :8000

# 杀死进程
kill -9 进程 ID

# 或使用其他端口
python3 -m http.server 8080
```

### 问题 2: 无法局域网访问

1. 检查防火墙设置
2. 确认设备在同一 WiFi
3. 尝试关闭防火墙

### 问题 3: GitHub Pages 404

1. 确认 gh-pages 分支存在
2. 等待 1-2 分钟同步
3. 检查文件名大小写
